SPIndleframe · Serialized Process Index

Every function you own, in one addressable index.

SPIndlebox empirically decomposes codebases — Python, TypeScript, Go, Rust, Bash, Java — into an SPI: a typed, nested, data-driven index of every function. So nothing gets rebuilt that already exists, and migrating to another language becomes mechanical instead of heroic.

Why an index

  
Your functions are data. Treat them like it.

  
A data architect wouldn't run a warehouse where nobody knows what
  tables exist, what their schemas are, or which three teams built the same one.
  Most codebases are exactly that — for functions. The SPI is the catalog:

  

    
Itemized functions
      
Every function, method, closure and lambda is an item with a sticky ordinal
      and a dotted address: src.utils.io.Reader.read_lines

    
Cross-language signature classes
      
Types normalize to a small core vocabulary, so the same shape in different
      languages lands in the same class — dedup works across repos and languages.

    
Context normalization
      
Every item gets a canonical ctx in → ctx out
      adapter. Heterogeneous functions become uniform, chainable ops.

    
State-capture classes
      
Each item is classified pure / reads / mutates / consumes —
      fn · Fn · FnMut · FnOnce — before you ever port it.

    
Homogeneous op arrays
      
Groups partition members by signature class: one element type per array,
      enforced by spindlebox validate, like any good schema.

    
Typed pipelines
      
pipeline define type-checks stage N → N+1 at
      definition time. Broken chains are refused before they run.

Everything is data

  
Adding a language is a JSON file, not a fork.

  
Input languages are extraction profiles; output languages are
  emit profiles. A profile carries the grammar wheel, node-role tables, a type
  table, identifier rules. Java — indexing and generation — exists entirely as
  data: there is no java_*.py anywhere, and a test asserts it.

  
// emit_profiles/java.json — the parts that made real repos compile
"simple": {"str": "String", "i64": "Long", "bool": "Boolean", "unit": "Void"},
"list_bases": ["List", "ArrayList", "Collection"],
"reserved_member_names": ["clone", "equals", "hashCode", "toString", "wait"],
"unique_result_binding": true

Evidence

  
Hardened against real code, in public.

  
A continuous test ladder runs spindlebox against 15 real repositories
  — from fixtures up to pydantic, fastapi, zod and gin — scoring every run: index
  cleanly, validate, generate, compile. Every red became a GitHub issue, a fix,
  and a live re-verification. 12 issues found → fixed → verified; 47 scored runs;
  all tiers green at v1.0.0.

  

    

      T0 fixtures
      T1 small real repos
      T2 click · express · serde_json · flask
      T3 fastapi · pydantic · zod · gin
    

    

    

  

  
Each dip is a tier opening and finding
  something real — duplicate ordinals in pydantic, a Rust keyword collision, a Java
  clone() illegally hiding Object.clone().
  Each recovery is a verified fix. The full ledger lives in the
  issue tracker.






  
Start

  
Index something in the next two minutes.

  
git clone https://github.com/moonsoup/spindlebox && cd spindlebox
pip3 install -e ".[dev]"
spindlebox index ~/code/yourproject     # → .spi/index.json + registry
spindlebox validate --strict
spindlebox report typing-health
